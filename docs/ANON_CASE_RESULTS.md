# 匿名案例结果格式

## 目标
在真实案例积累还不多时，先把案例展示结构标准化，后续可以直接替换为匿名真实案例。

---

## 案例模板
### 案例标题
例如：`某 VPS 单实例部署上线前体检`

### 部署背景
- 部署环境：VPS / NAS / 家用主机 / 团队试点
- 能力接入：消息 / 浏览器 / 节点 / skill / cron
- 当前阶段：刚上线 / 准备扩容 / 客户交付前

### 核心发现
- 风险评分
- 风险等级
- 最高优先级问题
- 是否建议立即整改

### 典型问题
- 公网暴露
- 权限过宽
- 日志 / 审计缺失
- 节点 / skill 风险
- 成本与告警观察点缺失

### 输出结果
- 给出正式报告
- 给出整改优先级
- 给出基线建议
- 是否升级到 Pro / 团队版

---

## 占位展示原则
1. 不暴露真实客户身份与敏感信息。
2. 重点展示“问题类型 + 输出结果 + 后续动作”。
3. 先证明你能交付什么，再逐步积累更强案例证据。
