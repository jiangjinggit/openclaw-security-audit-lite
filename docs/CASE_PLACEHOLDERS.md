# 首批案例占位

## 案例 1：独立开发者上线前检查（占位）
- 部署环境：VPS / 单实例
- 主要问题：公网暴露、日志缺失、权限边界不清
- 输出结果：风险评分 + 整改优先级 + 基线建议

## 案例 2：给客户交付前体检（占位）
- 部署环境：客户私有化试点
- 主要问题：交付前缺少正式报告与风险说明
- 输出结果：标准版体检报告 + Pro 修复方向

## 案例 3：小团队扩容前风险确认（占位）
- 部署环境：多能力接入 / 准备接更多节点
- 主要问题：skill / 节点 / 浏览器能力组合风险
- 输出结果：风险分组 + 扩容前收敛建议

## 使用原则
在真实案例不足前，先用占位模块说明“未来会展示什么类型的案例”；
拿到首批样本后，再逐步替换为匿名真实案例。
