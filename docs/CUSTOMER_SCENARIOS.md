# OpenClaw 样例客户 / 使用场景

## 场景 1：独立开发者上线前检查
### 当前状态
- 已经把 OpenClaw 跑在 VPS / NAS / 家用主机上
- 准备接真实账号、真实任务或更多能力

### 最关心的问题
- 现在这样到底安不安全？
- 是否已经可以进入真实使用阶段？
- 最先该修哪几个点？

### 最适合的购买路径
免费预检 → 标准版体检报告 → Pro 修复版

---

## 场景 2：给客户交付前做一次体检
### 当前状态
- 已经帮客户完成初步部署
- 需要一份更像交付物的风险说明和整改建议

### 最关心的问题
- 怎么把当前风险讲清楚？
- 怎么给客户一份可汇报、可归档的结果？
- 哪些问题必须先收敛再交付？

### 最适合的购买路径
标准版体检报告 → Pro 修复版 → 团队 / 企业版

---

## 场景 3：小团队试点前做风险确认
### 当前状态
- 已经在团队内部试跑 OpenClaw
- 准备接更多节点、账号、消息渠道或浏览器能力

### 最关心的问题
- 现在扩容会不会把风险一起放大？
- 是否已经具备最小安全边界？
- 需要先做哪些收敛动作？

### 最适合的购买路径
免费预检 → 标准版体检报告 → 团队 / 企业版

---

## 场景 4：多能力重度用户做阶段性体检
### 当前状态
- 已接入消息、浏览器、文件、节点、skill
- 已进入“能跑但复杂度明显上升”的阶段

### 最关心的问题
- 哪些组合点最容易失控？
- 哪些权限和能力边界最需要先收敛？
- 是否需要进入持续巡检阶段？

### 最适合的购买路径
标准版体检报告 → Pro 修复版 → 持续巡检 / 团队版
