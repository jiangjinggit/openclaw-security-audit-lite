# OpenClaw 首页一页式收口

## 目标
把首页进一步压成更像公开销售页的结构，而不是持续增长的信息墙。

---

## 首页最该优先传达的 4 件事
### 1. 这是给谁的
给已经自托管 OpenClaw、准备上线或准备扩容的人。

### 2. 先卖什么
先卖：
**299 元首单体检报告**

### 3. 用户会拿到什么
- 部署风险评分
- 风险分组
- 整改优先级
- 正式报告
- 轻量运维观测建议

### 4. 用户下一步怎么做
- 免费预检
- 再升级 299 元体检报告
- 再升级 Pro / 团队版

---

## 首页结构建议
1. Hero：一句话讲清产品和首单动作
2. 首单推荐：299 元体检报告
3. 适合谁 / 典型风险场景
4. 套餐对比
5. 样例输出 / 样例报告
6. 交付节奏
7. FAQ
8. 最后 CTA

---

## 收口原则
1. 首屏不要再讲太多平行概念。
2. 首屏优先卖“免费预检”和“299 元首单体检报告”。
3. 其余模块服务于解释、筛选、增强信任。
